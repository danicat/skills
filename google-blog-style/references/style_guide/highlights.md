frivolous.\n- Don't pre-announce anything in
documentation.\n- Use descriptive link text.\n- Write accessibly.\n- Write for a global audience.\n\n\n## Language and grammar\n\n- Use second person: "you" rather than
"we."\n- Use active voice: make clear who's performing
the action.\n- Use standard American spelling and
punctuation.\n- Put conditions before instructions,
    not after.\n- For usage and spelling of specific words, see
the word list.\n\n\n## Formatting, punctuation, and organization\n\n- Use sentence case for document
titles and section headings.\n- Use numbered lists for sequences.\n- Use bulleted lists for most other lists.\n- Use description lists for pairs of related
pieces of data.\n- Use serial commas.\n- Put code-related text in code font.\n- Put UI elements in bold.\n- Use unambiguous date formatting.\n\n\n## Images\n\n- Provide alt
text.\n- Provide high-resolution or vector
images when practical.\n
